q-vid and HUD Components

QHTML 6.9.8 adds the q-vid animation stack, HUD visual assets, and related component tests. The q-vid player is built as source, painter, and player components so decoding, timing, and canvas painting stay separated.

q-vid Component Stack

q-vid-source

Owns the decoder worker, multi-URL frame cache, and frame data API. It emits `fileLoaded`, `frameReady`, `frameData`, and `frameNotReady`.

q-vid-painter

Owns playback timing. It emits readiness and frame-data intent, accepts bytes, and emits `frameReadyToPaint` after a frame is accepted.

q-vid-player

Public canvas component. It mediates all source/painter traffic, binds player properties into the painter, paints frames, and emits `started`, `stopped`, and `stepped`.

Basic Playback

q-import { q-components/q-vid-player.qhtml }

q-vid-player hudPlayer {
  src: "tools/assets/hud-extend.qvid"
  frameDuration: 16
  repeat: true
  scale: 1
  onstepped(currentFrame) {
    console.log("painted frame", currentFrame);
  }
}

button { text { Start } onclick { hudPlayer.start(); } }
button { text { Stop } onclick { hudPlayer.stop(); } }
button { text { Step } onclick { hudPlayer.step(); } }

Included Files

  • dist/q-components/q-vid.qhtml: source, painter, and player implementation.
  • dist/q-components/q-vid-player.qhtml: import wrapper for user pages.
  • tools/q-vid/: browser encoder and standalone q-vid player/debugger.
  • tools/assets/hud-extend.qvid and related HUD images: sample visual assets.
  • test/visual/q-vid-player.html and test/visual/q-vid-animations.html: manual visual smoke pages.

Related Visual Components

The release also includes `q-tech-button`, `q-control-arm-screen`, and updated animation group tests for HUD-style interfaces. Import them from `q-components.qhtml` or directly from `dist/q-components/` when building focused pages.
